# Lua配置工具
# 设计目的
设计Excel到lua配置的配置工具。Excel侧方便策划进行配置，lua提高代码侧读取效率
# 具体设计
## 数据要求
- 使用Python为主语言，读取excel数据。导出lua数据，后续对接到Unlua中，供UE引擎开发游戏进行读取。
## UI界面
- 需要UI交互界面，
    - 搜索筛选列表：可通过输入框输入字符对文件名含有该字符的文件进行筛选。
        - 默认展示所有xlsx文件
        - 输入字符后会刷新，仅展示符合条件的文件。
            - 选择需要遍历该文件夹以及所有的子文件夹，将所有的xlsx按照文件名罗列到列表中。
        - 可以单击选中文件，右键唤出菜单后，选择【导出】执行导出操作。
        - 可以双击文件打开，也可以通过单击选中，右键唤出菜单，选择打开。
    - 按钮1：用于配置文件路径。选择电脑上的文件夹作为导入文件的路径，按下后刷新搜索筛选列表。
    - 按钮2：选择导出文件的路径。
## 数据导出规则
- excel第一列第一行定义导出lua的名称
    - 配置示意，测试表:TestConfig
        - 实际输出文件：TestConfig.lua
    - 输入多张表，多个Sheet时，会把输出结果进行拼接。
        - 例如表a.xlsx有【测试表:TestConfig】表b.xlsx有【测试表分表:TestConfig】则会将两张表第三行起的数据按照导出规则进行导出。
        - 多表合并时，列结构按照
- excel第一列除了第一行，用于定义字段备注和导出字段名，备注名和字段名使用"|"分隔
    - 填写示例【测试用Const|TestConst】
    - 导出的字段名为TestConst
- excel第一列从第三行起，值代表是否导出
    - 为0/空时，不导出本行
    - 为1，导出本行
- 第三行起为正式数据，标记为需要导出时进行导出。支持以下格式，每列的格式由第二行配置的值定义
    - Int，整形
    - Float，浮点型
    - String，文本型
    - Const，枚举型，填写时填写字符串，导出时，导出对应的序号
        - 定义填写示意Const(Typename1=1,Typename2=2)
        - 该列填写示意，Typename1.
        - 导出示意，1
    - Array，数组型
        - 填写示意，[1,2,3]
    - Dic，字典型，支持Int，String作为主键，上述其他值作为值进行输出
        - 定义填写示意，Dic(String:Int) ，Dic(String:Float)，Dic(String:Array)
    - List，集合型，支持上述任意类型的组合
        - 定义示意，List(Int)，List(Float)，List(String)，List（Array），List(List(Int))
    - Ignore，不导出的列
- 所有表都会有ID列，字段名为ID，
    - ID列作为主Key，默认不可重复。若发生重复，则需要检查是否有SubID字段，ID重复时，SubID必须不重复。
        - 默认格式为
        ```cpp
local TestConfig = {
    [1001] = {Name = "新手剑", Atk = 15, Type = 1},
    [1002] = {Name = "铁盾",   Def = 20, Type = 2},
    [1003] = {Name = "皮甲",   Def = 8,  Type = 2},
}
```
        - 存在子ID时，格式为
        ```cpp
local TestConfig = {
    [1001] = {
        [1] = {Name = "新手剑",   Atk = 15},
        [2] = {Name = "新手剑+1", Atk = 20},
        [3] = {Name = "新手剑+2", Atk = 28},
    },
    [1002] = {
        [1] = {Name = "铁盾",     Def = 20},
    },
}
```
    - ID的类型可以为String或者Int类型

## 数据检查
导出时需要执行以下检查
- 格式检查，检查各个字段填写的值是否符合第二行的配置
- Const值不在定义范围内时，需要提示报错。
- 字段名重复报错
- ID，SubID检查
